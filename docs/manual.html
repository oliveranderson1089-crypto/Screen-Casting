scrcpy 投屏手册

在电脑上显示并操作 vivo 手机。日常只需点桌面图标「手机投屏」—— 下面是启动之后能用的全部按键与命令。

  • 设备 vivo V2231A · Android 16
  • scrcpy 3.3.4
  • adb 37.0.0
  • 连接 无线 · 局域网 5555 端口

启动 三种方式，效果相同

日常用法

按 Super 搜「投屏」，点「手机投屏」图标。 插着数据线就走 USB（画质最好），没插线自动走无线。

右键图标可直接选通道：USB 投屏 · 无线投屏 · 重新打通无线 · 断开无线连接。

终端命令作用
phone-cast.sh 自动选择通道，日常用这个
phone-cast.sh usb 强制走 USB，码率 24M
phone-cast.sh wifi 强制走无线，码率 12M。记录的 IP 连不上会自动扫网段找手机
phone-cast.sh desktop 独立桌面模式：电脑上开一块独立 Android 桌面，手机主屏可休眠省电（非镜像）
phone-cast.sh find 只扫描、更新 IP 记录，不启动投屏。想手动刷新地址时用
phone-cast.sh setup 手机重启后插线跑一次，重新打通无线
phone-cast.sh off 断开无线 adb 连接

脚本已默认带上关屏参数：启动即关闭手机屏幕，省电、防偷看， 电脑这边照常显示和操作。想让手机屏幕亮起来按 Alt+Shift+O。

两种模式 镜像 / 独立桌面

默认是镜像——电脑上看到手机当前界面。另有一个可选的独立桌面模式， 右键图标选「独立桌面」或跑 phone-cast.sh desktop。两者互不影响，随时切换。

对比镜像（默认）独立桌面（desktop）
电脑上看到 手机当前界面 你选定的一个应用，独立运行
手机主屏 关闭省电，但系统保持唤醒 可正常锁屏、休眠，真省电
手感 和手机同步，来电消息都在眼前 像多开一台设备，手机各干各的
适合 日常操控、演示 挂个应用长跑、不想占用手机

独立桌面靠 --new-display 开一块虚拟屏，需要 Android 11+（越新支持越好）。 启动时会弹框让你选一个应用在虚拟屏里打开（并记住上次的选择）。 之所以要指定应用，是因为 vivo 等国产 ROM 的辅助屏系统桌面时有时无、常是白板， 直接开一个应用最稳。关闭窗口时虚拟屏销毁，手机主屏不受影响。 窗口里的快捷键和镜像模式完全一样。

无线模式的省电前提

手机深度休眠会断开 WiFi，连带掐断投屏会话。所以想「挂个应用长跑」并离开， 建议用 USB 连接（USB 下手机休眠不影响 adb）；无线更适合人在旁边的短时使用。

无线连接是怎么回事 换网络会自动适应

前提：同一局域网

无线投屏走的是手机的局域网 IP，所以手机和电脑必须在同一个网络： 连同一个 WiFi、或电脑连手机开的热点，都可以。手机用蜂窝数据、连别的 WiFi，则不行。 跨公网远程投屏做不到——那是私有地址，公网无法路由。

换了网络 / IP 变了会自动找回

路由器可能给手机分配新 IP（换 WiFi、重连、开热点都会）。记录的地址连不上时， 脚本会自动扫描当前网段，靠硬件序列号认出你的手机 （网段里别人的调试设备序列号对不上，会跳过），找到后更新记录。 整个过程几秒钟，你不用管。

唯一的前提是手机 WiFi 醒着。手机深度休眠时 Android 会主动断开 WiFi （连 ping 都不通，这是省电机制，非脚本问题）。扫不到时， 先点亮一下手机屏幕让 WiFi 恢复，再试即可。

遇到问题 按现象查

画面突然变黑，但窗口还在

手机闲置满 10 分钟真休眠了。按 Alt+Shift+O 唤醒即可，不用碰手机。

你在电脑上操作时计时器会不断重置，所以正常使用中碰不到这个。想彻底避免，只能在手机 设置 → 显示与亮度 → 自动锁屏 里手动调长 —— vivo 拦截了从电脑改这个设置。

连不上手机 / 提示连接失败

按这个顺序排查：

1. 先点亮手机屏幕。手机深度休眠时 WiFi 会断，扫描找不到（最常见、也最容易解决）。亮屏后 WiFi 恢复，再试即可。

2. 手机换了 IP？不用管。脚本会自动扫网段靠序列号找回——这是设计好的，无需手动干预。如果想手动刷新，跑 phone-cast.sh find。

3. 手机重启过。adb tcpip 不持久，重启后无线通道彻底失效，扫描也救不回来。这时必须插 USB 线、右键图标选「重新打通无线」。

4. 不在同一网络。确认手机和电脑连的是同一个 WiFi（或电脑连了手机热点），且路由器没开「AP 隔离」。

拔数据线时画面断了

正常现象。拔线瞬间 adb 会重新枚举设备，把正在跑的无线会话一起带断。无线通道本身没坏，重新点一次图标就行。

USB 连接时手机没反应

下拉手机通知栏，把 USB 用途设为「管理文件」—— 这是 vivo 对标准「文件传输」的叫法，菜单里没有叫「传输文件」的选项。

还要确认开发者选项 → USB 调试是开着的，并且首次连接时在手机上点了「一律允许」。

画面卡顿、延迟明显

无线走的是 12M 码率，受 WiFi 信号影响。改善顺序：靠近路由器 → 改用 USB 连接（24M 码率，延迟最低）。

如果是花屏或画面凝固，按 Alt+Shift+R 重置视频流。

同时在打 APK 包，两边互相断连

已经处理过了：终端里的 adb 和投屏脚本统一走 SDK 那份 37.0.0，不会再和系统自带的 34.0.5 抢 5037 端口。

如果仍然遇到，检查是不是有脚本硬写了 /usr/bin/adb 的绝对路径。
