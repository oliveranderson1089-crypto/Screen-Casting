# Screen-Casting

把安卓手机投屏到 Linux 电脑，基于 [scrcpy](https://github.com/Genymobile/scrcpy)。

一个封装脚本 + 桌面图标，解决日常投屏的几个痛点：**一键在 USB / 无线间自动切换**、**手机黑屏省电但电脑照常操作**、**换了 WiFi 或 IP 变了自动重连**。在 vivo（OriginOS / Android 16）上实测，逻辑对任意安卓机通用。

> 完整图文操作手册见 [`docs/manual.html`](docs/manual.html)（含全部快捷键，浏览器打开）。

---

## 亮点

| 能力 | 说明 |
|------|------|
| **自动选通道** | 插着数据线走 USB（画质最好），没插线走无线，不用手动指定 |
| **手机黑屏、电脑照常用** | 启动即关闭手机屏幕面板省电、防偷看；scrcpy 用独立虚拟显示取画面，物理面板断电不影响 |
| **两种模式** | 镜像（默认）或独立桌面（`desktop`）——后者在电脑上开一块独立 Android 桌面，手机主屏可彻底休眠 |
| **换网络自动找回** | 手机换了 WiFi、开了热点、路由器重新分配了 IP，脚本靠**硬件序列号**在当前网段里把手机认出来，自动更新记录 |
| **认得是自己的手机** | 网段里有别人的调试设备也不会连错——用序列号校验，不匹配就跳过 |
| **出错有提示** | 从桌面图标启动时没有终端，失败会弹图形对话框并给出排查步骤 |

## 依赖

```bash
sudo apt install scrcpy adb      # Ubuntu / Debian
```

- **scrcpy ≥ 1.25**（关屏、虚拟显示等需要较新版本；开发用 3.3.4）
- **adb**（Android Platform Tools）
- 手机需开启**开发者选项 → USB 调试**

## 安装

```bash
git clone https://github.com/oliveranderson1089-crypto/Screen-Casting.git
cd Screen-Casting
./install.sh
```

`install.sh` 会按当前路径生成桌面图标、注册到应用列表（GNOME 下自动加进应用网格）。之后按 `Super` 搜「投屏」即可。也可以不装图标，直接命令行运行 `./phone-cast.sh`。

## 首次打通无线

无线连接需要先用 USB 线引导一次：

```bash
./phone-cast.sh setup      # 插着 USB 线运行，把无线通道打通并记住手机
```

之后拔线也能直接无线投屏。**手机重启后**无线调试会失效，需再跑一次 `setup`。

## 用法

```bash
./phone-cast.sh            # 自动：有线走 USB，无线走 WiFi（日常用这个）
./phone-cast.sh usb        # 强制 USB，码率 24M
./phone-cast.sh wifi       # 强制无线，码率 12M；连不上会自动扫网段找手机
./phone-cast.sh desktop        # 独立虚拟屏：弹框选一个应用在里面打开，手机主屏可休眠省电
./phone-cast.sh desktop <包名>  # 直接打开指定应用（跳过选择框，如 com.android.chrome）
./phone-cast.sh find       # 只扫描、更新 IP 记录，不启动投屏
./phone-cast.sh setup      # 插线重新打通无线（手机重启后需要）
./phone-cast.sh off        # 断开无线连接
```

右键桌面图标可直接选：USB 投屏 / 无线投屏 / 独立桌面 / 重新打通无线 / 断开无线。

### 镜像 vs 独立桌面

| | 镜像（默认） | 独立桌面（`desktop`） |
|---|---|---|
| 电脑上看到的 | 手机当前界面 | 你选定的一个应用，独立运行 |
| 手机主屏 | 关闭省电，但系统保持唤醒 | 可以正常锁屏、休眠，**真省电**（无线见下方注意） |
| 手感 | 和手机同步，接电话消息都在 | 像多开一台设备，手机各干各的 |
| 适合 | 日常操控手机、演示 | 挂个应用长时间跑、不想占用手机 |

独立桌面靠 `--new-display` 开一块虚拟屏，需要 Android 11+（越新支持越好）。**启动时会弹框让你选一个应用**在虚拟屏里打开（并记住上次的选择），也可以命令行直接给包名。之所以要指定应用，是因为很多国产 ROM（含 vivo）的辅助屏系统桌面时有时无、常是白板，直接开一个应用最稳。关闭窗口时虚拟屏销毁，手机主屏不受影响。

> ⚠️ **无线模式的省电有个前提**：手机深度休眠会断开 WiFi，连带掐断投屏会话。所以「挂个应用长跑」这类离开场景建议用 **USB** 连接（USB 下手机休眠不影响 adb）；无线更适合人在旁边的短时使用。

## 常用快捷键

投屏窗口里，`MOD` 默认是**左 Alt** 或**左 Super**：

| 快捷键 | 作用 |
|--------|------|
| `MOD` + `O` | 关闭手机屏幕（投屏继续，省电防偷看） |
| `MOD` + `Shift` + `O` | 点亮手机屏幕（休眠后用它恢复） |
| `MOD` + `H` | 主页　`MOD` + `B` 返回　`MOD` + `S` 任务切换 |
| `MOD` + `F` | 全屏　`MOD` + `W` 去黑边　`MOD` + `G` 1:1 缩放 |
| `MOD` + `C` / `V` | 手机→电脑复制 / 电脑→手机粘贴 |
| 拖文件进窗口 | 传到手机（APK 直接安装） |

完整列表见 [`docs/manual.html`](docs/manual.html)。

## 无线连接的前提

无线走的是手机的**局域网 IP**，所以：

- ✅ 手机和电脑连同一个 WiFi，或电脑连手机热点
- ❌ 手机用蜂窝数据、连别的 WiFi、跨公网——私有地址公网无法路由
- ⚠️ **手机深度休眠时 WiFi 会断**（这是 Android 省电机制，非本工具问题），扫不到时先点亮一下手机屏幕再试

## 排查

| 现象 | 处理 |
|------|------|
| 连不上 | ①先点亮手机屏幕　②IP 变了不用管，会自动找回　③手机重启过要插线 `setup`　④确认同一网络 |
| 画面突然变黑 | 手机闲置休眠了，按 `MOD` + `Shift` + `O` 唤醒 |
| 画面卡顿 | 靠近路由器，或改用 USB（延迟最低）；花屏按 `MOD` + `Shift` + `R` 重置 |
| USB 没反应 | 通知栏把 USB 用途设为「文件传输」（vivo 叫「管理文件」），并确认已开 USB 调试、点过「一律允许」 |

## 说明

- 脚本优先使用 `~/AppStore/toolchains/android-sdk` 下的 adb，找不到则回退系统 `adb`——避免多个 adb 版本抢 5037 端口。克隆到其他机器无此目录时自动走系统 adb。
- 运行时状态（无线 IP、设备序列号）存在 `~/AppStore/data/phone-cast/`，已在 `.gitignore` 中排除，不进版本库。
